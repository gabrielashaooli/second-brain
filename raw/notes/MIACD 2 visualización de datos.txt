## Contenido principal — teoría del color y su aplicación en visualización

- Propósito del color: mejorar comprensión, resaltar patrones y guiar interpretación de datos.
- Consideraciones de audiencia: daltónicos (aprox. 9% con daltonismo rojo-verde), diferencias culturales en semántica de color; conocer público objetivo antes de elegir paleta.
- Parámetros del color (rueda de color): matiz (hue), saturación y luminosidad (lightness). Explicación de cómo cada parámetro altera percepción:
  - Matiz: cambia la tonalidad (rojo, azul, verde, etc.).
  - Saturación: intensidad del color; 0 = gris, 100 = color puro.
  - Luminosidad: mezcla con blanco/negro; mayor luminosidad = más claro.
- Temperatura del color: colores cálidos percibidos como cercanos/intimos; colores fríos como distantes/analíticos.
- Significados semánticos y ejemplos: rojo = alerta/poder; azul = seriedad/confianza; amarillo = optimismo; negro = elegancia/ausencia; ejemplos de marcas bancarias (Banorte rojo, BBVA azul).
- Herramientas y referencias: Paletón (generador de paletas), sistemas Pantone (estandarización), pruebas visuales para daltonismo, recursos para cartografía.
- Uso ético del color: advertencia sobre capacidad de enfatizar u ocultar información mediante color (responsabilidad del presentador).

## Paletas y relaciones entre colores

- Color clave: color base o dominante (marca personal o corporativa). Usarlo para resaltar series de datos principales.
- Estrategias de paleta:
  - Análogos: colores contiguos en la rueda (p.ej., naranja + rojo + amarillo) para indicar relación sin juicio de valor.
  - Complementarios: colores opuestos para contrastes y juicios (p.ej., naranja vs. azul para enfatizar contraposición positivo/negativo).
  - Complementario dividido (split-complementary): color base + dos colores cercanos al complementario.
  - Dobles complementarios, tetrádicos y rampas (secuenciales y divergentes) para datos continuos y cuantitativos.
- Recomendaciones: para datos categóricos variar tono; para datos continuos variar saturación/luminosidad; usar gris como color de de-emphasis (comodín) para elementos menos importantes.

## Texto y énfasis visual

- Uso de color y negritas en texto para guiar la mirada: coloriza solo lo esencial; la tipografía y color influyen en qué lee primero la audiencia.

## Tipos de gráficos — selección y buenas prácticas

- Principio general: elegir el tipo de gráfico según el propósito y el público; evitar sobrecargar con elementos innecesarios.

Se examinaron (con ventajas, limitaciones y tips de diseño):
- Series temporales / líneas: mostrar tendencias y forecast; incluir contexto; atenuar series secundarias; evitar múltiples líneas excesivas.
- Gráficos de área y área apilada: útiles para parte/total y evolución; cuidado con solapamientos y pérdida de detalle; transparencia ayuda.
- Barras (verticales, horizontales, agrupadas, apiladas, 100% apiladas): recomendadas casi siempre para comparación categórica; alinear etiquetas, ordenar por importancia (mayor → menor), mantener correcta separación (~30% regla aproximada), usar baseline en cero; colocar etiquetas dentro de barras para lectura más rápida.
- Boxplot (diagrama de caja y bigotes): útil para ver distribución y outliers; es apto para audiencia estadística; alternativas: histogramas, swarm plots, violin plots, dot plots.
- Burbujas (bubble charts): incorporar tres dimensiones (X, Y, tamaño) y color para una cuarta; evitar si la tercera dimensión no aporta variación relevante.
- Viñeta (bullet chart): útil para comparar con objetivo/umbral; compacta y efectiva para KPIs.
- Tablas: útiles para audiencias que necesitan detalle; en vivo reducen atención al orador; recomendaciones de diseño: eliminar bordes innecesarios, alinear números, usar minigráficos y heatmaps para facilitar lectura.
- Diagramas de flujo (flowcharts): indispensable para procesos; evitar saturación de color; usar colores solo con propósito.
- Gráficos de puntos / scatter plots: útiles para relaciones y outliers; dot plots y connected dot plots para comparar valores discretos.

## Observaciones y recomendaciones prácticas dadas en clase

- Conocer siempre al público objetivo antes de elegir colores, grafico y nivel de detalle.

- Incluir contexto y anotaciones cuando la audiencia no conoce los datos.
- Usar tablas solo cuando el público requiere detalle; en presentaciones en vivo, preferir visuales claras.
